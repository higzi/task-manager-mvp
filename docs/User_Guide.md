# 3. Руководство пользователя

---

## 3.1 Как начать работу

Для начала работы с приложением:

1. Откройте веб-приложение в браузере  
2. Дождитесь загрузки интерфейса  

> В текущей версии MVP регистрация и авторизация отсутствуют.  
> Пользователь может сразу приступить к работе с задачами.

---

## 3.2 Как ориентироваться в интерфейсе

Интерфейс приложения состоит из следующих основных элементов:

- Список задач — отображает все созданные задачи  
- Форма создания/редактирования — используется для добавления и изменения задач  
- Элементы управления (кнопки):
  - создание задачи  
  - редактирование  
  - удаление  
- Отображение статуса задачи  

Навигация осуществляется через один основной экран (SPA), без переходов между страницами.

---

## 3.3 Основные действия пользователя

### Создание задачи
Для создания задачи необходимо:

1. Заполнить форму (название, описание)  
2. Нажать кнопку создания  
3. Задача появится в списке  

---

### Редактирование задачи
Для редактирования:

1. Выбрать задачу  
2. Нажать кнопку редактирования  
3. Изменить данные  
4. Сохранить изменения  

---

### Удаление задачи
Для удаления задачи:

1. Найти задачу в списке  
2. Нажать кнопку удаления  
3. Подтвердить действие (если предусмотрено)  

Задача будет удалена из системы.

---

## 3.4 Как организовывать данные

В текущей версии MVP доступны базовые способы организации:

- Использование статусов задач:
  - todo  
  - in-progress  
  - done  

- Логическое разделение задач по:
  - приоритету (вручную пользователем)  
  - содержанию (название и описание)  

> Проекты, теги и категории в MVP не реализованы, но могут быть добавлены в будущем.

---

## 3.5 Как искать и фильтровать

В MVP предусмотрены базовые возможности:

- Просмотр всех задач в списке  
- Фильтрация по статусу (если реализована в интерфейсе)  
- Визуальное разделение задач по состоянию  

> Полноценный поиск по ключевым словам может отсутствовать или реализован частично.

---

## 3.6 Как работают уведомления

В текущей версии MVP:

- Уведомления отсутствуют  
- Система не отправляет напоминания или оповещения  

> В будущем возможно добавление:
- push-уведомлений  
- email-уведомлений  
- напоминаний о дедлайнах  

---

## Примечание

Данное руководство описывает функциональность уровня MVP и может быть расширено по мере развития приложения.
